# GitHub Pages Deploy Skill

Prepare static websites for GitHub Pages.

Rules:
- Keep the site static.
- Use index.html at the repo root unless there is a clear reason not to.
- Use relative asset paths.
- Do not require a server.
- Do not require build tooling unless necessary.
- Include a clear README.md.
- Include preview instructions.
- Include GitHub Pages deployment instructions.

Deployment notes:
- GitHub Pages can serve static sites from a branch.
- Keep the first version simple.
- Use no backend.
- Use placeholder CTA links until a booking tool exists.
